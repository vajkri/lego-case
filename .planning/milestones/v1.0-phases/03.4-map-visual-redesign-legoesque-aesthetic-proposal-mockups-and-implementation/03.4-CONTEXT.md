<decisions>
## Implementation Decisions

### Mockup-first approach
- Create a standalone HTML mockup file (like `stop-marker-proposals.html` and `styleguide.html`)
- **Section 1:** 2-3 full map scene directions showing different overall aesthetic approaches
- **Section 2:** Per-element proposals (2-3 options each) for trees, mountains, buildings, windmill, pond, fence, sky, grass
- User reviews mockup, selects direction and per-element choices, then implementation proceeds
- Location: `.planning/assets/mockups/map-redesign-proposals.html`

### Visual constraints from assumptions discussion
- **Sky and grass:** User is happy with current colors. Changes need to prove themselves in the mockup — don't assume they'll be accepted
- **Road:** User is happy with current road treatment. May try variations in mockup but road is "prove it" territory
- **Color changes:** Must be contrast-aware. Don't blindly swap to design tokens if it damages readability. Permitted to register new colors into the design system if needed
- **Style target:** Match the BrickMarker stop nodes and minifig head progress track visual language — LEGO-inspired flat illustration, not hyperrealistic brick rendering

### Creative freedom granted
- **New vs restyled elements:** Full discretion — can add new decorative elements or just restyle existing ones
- **Decorative additions:** Free to propose minifig characters, baseplate edges, flowers, signposts, etc.
- **Aesthetic direction:** User doesn't have a strong preference — wants to see creative proposals and react
- **Use frontend-design skill** for the mockup creation — go bold and creative

### Late-phase polish tasks (after main redesign)
1. **Ambient animations** (from todo: `2026-03-12-add-subtle-animations-to-map-elements.md`):
   - Sun: gentle pulse or slow rotation of rays (~8-10s period)
   - Clouds: slow horizontal drift (~20-40s period)
   - Windmill: continuous slow blade rotation (~6-8s period)
   - CSS `transform`/`opacity` only, respect `prefers-reduced-motion`
2. **Shadow cleanup** (from todo: `2026-03-12-clean-up-shadows-for-map-elements.md`):
   - Remove road hover-shadow (`translate(3,6)` drop shadow in RoadPath.tsx)
   - Audit all element shadows for organic grounding
3. **Car stop alignment:**
   - Car doesn't stop exactly at stop nodes — adjust `STOP_OFFSETS` in `RoadPath.tsx` after map/road updates

### Claude's Discretion
- Which aesthetic directions to propose in the mockup (instruction manual, box art, or other LEGO styles)
- How many element variations per type (2-3, as appropriate)
- Whether to propose changes to sky/grass/road (allowed in mockup, but flagged as "prove it")
- Exact new colors, tokens, or decorative elements to add
- Implementation details for ambient animations (CSS keyframes vs Motion)
- How to handle SVG complexity if adding stud patterns or brick details

</decisions>

<code_context>
## Existing Code Insights

### Reusable Assets
- `MapSvg.tsx` (`src/components/features/map/`): Primary target — 240 lines, contains `Tree()` and `Cloud()` helper components, inline SVG with defs (gradients), 11 element groups (sky, sun, clouds, mountains, buildings, grass, trees, pond, windmill, fence, road)
- `RoadPath.tsx` (`src/components/features/map/`): Road rendering — `ROAD_PATH_D`, `STOP_OFFSETS`, `CAR_START_OFFSET`, drop shadow + surface + edge + dashes. Road shadow removal is a late-phase task.
- `StopNode.tsx` / `BrickMarker`: Reference for LEGO SVG style — blocky shapes, favicon proportions, CSS `transition` for state changes
- `globals.css`: Design tokens — `@theme` block for colors, `:root` for rgba values and depth shadows
- Stud dot pattern CSS: `radial-gradient(circle at 12px 12px, rgba(255,255,255,0.06) 6px, transparent 6px)` with `background-size: 32px 32px`

### Established Patterns
- Inline SVG in React components (no external SVG files)
- `'use client'` on MapSvg (already present)
- Gradient `<defs>` for fills, hardcoded color values (not design tokens) in SVG elements
- Helper components for repeated elements (`Tree()`, `Cloud()`)
- Array `.map()` for repeated decorative elements (grass dots, building windows, fence posts)

### Integration Points
- `MapSvg.tsx`: Main file to modify (landscape elements)
- `RoadPath.tsx`: Shadow removal + potential road styling changes
- `RoadPath.tsx`: `STOP_OFFSETS` adjustment for car alignment
- `globals.css`: New design tokens if needed
- `CarElement.tsx`: No changes expected unless road path geometry changes
