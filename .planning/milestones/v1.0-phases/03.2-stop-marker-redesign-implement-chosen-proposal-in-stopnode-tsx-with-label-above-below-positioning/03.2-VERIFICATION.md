**Test:** Run `npm run dev`, open http://localhost:3000. Inspect each stop marker.
**Expected:**

- Stop 2: 2-brick grey stack (L1 full + L2 left half), label above grey background.
- Stop 3: 3-brick grey stack (L1 + L2 full), label below grey background (to the right of road).
- Stop 4: 4-brick grey stack (L1 + L2 full + L3 left half), label above.

#### 2. Hover and Focus Interactions

**Test:** Mouse over an unvisited stop (hover). Tab to a stop (focus).
**Expected:**

- Focus (Tab): 3px blue (#0055BF) outline on marker wrapper div with 4px offset. Stack scales to 1.08x.
**Why human:** CSS transform/filter visual behavior and outline appearance require browser rendering.

#### 3. Brick-Drop Entrance Animation

**Test:** Advance from stop 1 to stop 2 (ArrowRight). Watch stop 2 as car arrives.
**Expected:** The topmost brick (L2 left half) drops in from above with a spring-ease animation (~180ms). The lower L1 brick and baseplate do not animate — only the topmost `<g>` group plays the drop animation.
**Why human:** CSS animation playback with correct scope (topmost group only) requires visual inspection.

#### 4. Active-to-Visited Color Sweep

**Test:** Advance from stop 1 to stop 2. Watch stop 1 transition to visited.
**Expected:** Stop 1's bricks smoothly sweep from the active LEGO palette (red/green baseplate) to all-green bricks over ~300ms. This is the CSS `fill` transition on SVG rect elements.
**Why human:** CSS fill transition on SVG elements requires browser rendering to confirm.

#### 5. Brick Color Activation Timing

**Test:** Advance to a new stop. Watch carefully when the active stop's bricks switch from grey to colorful.
**Expected:** At approximately 80% through the car travel animation (around 1.12s into a 1.4s travel), the brick colors switch to the active LEGO palette — before the car fully arrives.
**Why human:** Timing-dependent visual behavior requires live observation.

#### 6. Reduced Motion

**Test:** Enable `prefers-reduced-motion: reduce` in Chrome DevTools (Rendering panel). Advance stops.
**Expected:** Brick-drop animation is absent. Bricks appear in final position immediately. All other visual states unchanged.
**Why human:** OS/browser media query behavior requires browser testing.

#### 7. Label Positioning (Road-Side Placement)

**Expected:** Stops 1, 2, 4, 5 have labels above the brick marker, offset 25px to the left of the road path. Stop 3 has its label below, offset 25px to the right. No label overlaps the road.
