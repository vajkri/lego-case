# Phase 03.3: Progress Track Redesign - Context

**Gathered:** 2026-03-14
**Status:** Ready for planning

<domain>
## Phase Boundary

Redesign the footer progress wizard in `PresentationFooter.tsx` — replace the generic numbered circles and thin connector lines with LEGO minifig heads (from Proposal #4 in `stop-marker-proposals.html`), thicker connectors with semantic colors, and sub-slide progress dots under the current stop. Remove the `MapProgressIndicator` corner widget (redundant). Does NOT include: stop marker changes on the map (03.2), overall map aesthetic (03.4), slide content (Phase 4), or road/path visual changes.

</domain>

<decisions>
## Implementation Decisions

### Minifig head step indicators
- Replace `w-8 h-8 rounded-full` buttons with inline SVG minifig heads (viewBox 34×40)
- Footer rendered size: `width="28" height="34"` per head
- Three base expressions only — no expression change on hover/focus:
  - **Default** (upcoming): grey head (#D4D4D8), classic LEGO smile (dot eyes + gentle curve mouth), grey features
  - **Current** (active): yellow head (#F5CD2D), excited grin (happy crescent eyes + wide open grin + cheek blush dots), dark features
  - **Visited** (complete): green head (#00A850), satisfied wink (one open eye + one wink arc + closed smile + stud checkmark), white features

### Head SVG anatomy (viewBox 34×40)
- **Stud**: x=11 y=2 w=12 h=4 rx=1 — flat cylinder, minimal border-radius
- **Head**: x=3 y=6 w=28 h=26 rx=8 — rounded rectangle, classic LEGO proportions
- **Neck**: x=9.8 y=32 w=14.4 h=4 rx=1 — wider than stud (90% of 16), same height and rx as stud
- Reference mockup: `.planning/assets/mockups/progress-track-states.html`
- Reference expression: `.planning/assets/Screenshot 2026-03-14 at 14.23.33.png` (classic LEGO smile for default)

### Hover & focus overlays (no expression change)
- Same pattern as Phase 03.2 stop node markers
- **Hover**: `scale(1.1)` + `drop-shadow(0 3px 6px rgba(0,0,0,0.2))` — CSS `transition: all 0.25s ease`
- **Focus**: `outline: 3px solid #0055BF`, `outline-offset: 4px` + `scale(1.08)` — `:focus-visible` only
- Expression stays the same — hover/focus are purely visual affordance overlays

### Connectors
- Thickness: `--border-default` (3px), up from current `h-0.5` (2px)
- Shape: `rounded-pill` caps
- Three semantic colors: `bg-lego-grey-pale` (default/upcoming), `bg-lego-red` (leading into current stop), `bg-lego-green` (between visited stops)
- **Left-to-right sweep** animation when a connector transitions to visited (~200ms)

### State transitions
- Head color fades ~200ms on state change (both outgoing and incoming heads transition simultaneously)
- Face features swap instantly — only the head/stud/neck color animates
- Connector fills with left-to-right sweep (~200ms) coordinated with head color fade
- Follows Phase 03.2 pattern: CSS transition on fill/background, no JS animation library

### Sub-slide progress dots
- Promote `SubSlideProgress` from `SlideFrame/SubSlideProgress.tsx` to `ui/SubSlideProgress.tsx` (shared component)
- Add `size?: 'default' | 'sm'` prop
- **default** (SlideFrame): 8px dots, 22px current pill, gap-1.5 (current behavior)
- **sm** (footer): 6px dots, 16px current pill, gap-0.5
- Only render under the **current** stop (not under visited/upcoming)
- Motion spring animation preserved for both sizes

### MapProgressIndicator removal
- Remove `MapProgressIndicator` component entirely (bottom-right corner dots on map view)
- Footer progress track is the sole progress indicator
- Remove import and usage from `MapCanvas.tsx`

### Footer behavior across views
- Progress track looks identical on map view and slide view — no visual adaptation
- Sub-slide dots update as slides advance within a stop
- All heads are always clickable — dispatch `JUMP_TO_STOP` regardless of current view (map or slide)
- No stop numbers displayed — too crowded at 28px scale (experimented and dropped)

### New design token
- Add `--color-lego-green-dark: #008040` to `@theme` in `globals.css` — used for visited head neck/stud fill

### Claude's Discretion
- Exact easing curve for the left-to-right connector sweep
- Whether to use CSS `background` transition or `linear-gradient` shift for the sweep effect
- How to structure the MinifigHead SVG component internally (inline vs extracted)
- Whether to keep the `MapProgressIndicator.tsx` file deleted or just unused
- Sub-slide dot gap value for `size="sm"` (mockup shows `gap-0.5` but may need tuning)

</decisions>

<specifics>
## Specific Ideas

- Default expression must be the **classic LEGO smile** — reference photo: `.planning/assets/Screenshot 2026-03-14 at 14.23.33.png`. Not a neutral/flat mouth. Small dot eyes and gentle curve smile.
- "The difference in states should be primarily the facial expression, not so much the color" — color reinforces but expression carries the meaning
- Hover/focus overlays follow the exact same pattern as stop node markers from Phase 03.2 — consistency across map markers and footer heads
- Stud on top of head has "too much border-radius" at rx=3 — use rx=1 for flat cylinder look
- Neck should be wider than stud and same height — proportional to real LEGO minifig head reference
- Stop numbers were experimented with (3 options: neck, badge-below, forehead) but all were dropped — too crowded at footer scale

</specifics>
