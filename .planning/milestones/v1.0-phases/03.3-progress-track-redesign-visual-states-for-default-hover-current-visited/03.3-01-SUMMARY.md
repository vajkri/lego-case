## Decisions Made
- SIZE_CONFIG object mapping (not conditional ternaries) for SubSlideProgress size variants — cleaner extensibility
- MinifigHead face expressions as separate named FCs (DefaultFace/CurrentFace/VisitedFace) — no CSS transition on face features, instant swap preserves clarity of state change
- MinifigHead is a Server Component — no hooks or event handlers needed, so 'use client' unnecessary
- jsdom normalizes hex colors to `rgb()` in inline styles — tests use dual-check pattern for robustness
