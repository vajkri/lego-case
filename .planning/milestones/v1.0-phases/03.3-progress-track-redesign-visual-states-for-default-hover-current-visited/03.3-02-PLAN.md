The new progress track section renders a flex row of stop steps separated by connectors. Each stop step is a vertical column: button (containing MinifigHead) on top, sub-slide dots or spacer below.

**Stop step button:**
- `<button>` wrapping `<MinifigHead state={headState} />`
- `headState` computed: `i < state.currentStop && state.visitedStops.includes(i)` => 'visited', `i === state.currentStop` => 'current', else => 'default'
- `onClick={() => handleJumpToStop(i)}`
- `aria-label={`Go to stop ${i + 1}: ${stop.label}`}`
- `aria-current={i === state.currentStop ? 'step' : undefined}`
- Hover/focus via local React state (`isHovered`, `isFocused`) per button — use `onMouseEnter`/`onMouseLeave` for hover, `onFocus`/`onBlur` for focus. ONLY one state pair needed for all 5 buttons — use an index-based approach: `hoveredIndex`/`focusedIndex` state (number | null).
- Button inline style function:
  - Default: `{ background: 'transparent', border: 'none', cursor: 'pointer', padding: '2px', outline: 'none', transition: 'all 0.25s ease' }`
  - Hover (when `hoveredIndex === i`): add `transform: 'scale(1.1)'`, `filter: 'drop-shadow(0 3px 6px rgba(0,0,0,0.2))'`
  - Focus (when `focusedIndex === i`): add `outline: '3px solid #0055BF'`, `outlineOffset: '4px'`, `borderRadius: '10px'`, `transform: 'scale(1.08)'`

**Sub-slide dots row (below each head):**
- Under the current stop ONLY: `<SubSlideProgress total={stops[state.currentStop].slides.length} current={state.currentSlide} size="sm" />`
- Under all other stops: a spacer `<div className="min-h-[8px]" />` to prevent footer height reflow
- Wrap in a container with `className="flex flex-col items-center"`

**Connectors (between stop steps):**
- Rendered for `i > 0` (4 connectors for 5 stops)
- A `<div>` with `className="flex-1 max-w-10 self-center"` (self-center aligns vertically with heads, not with dots row)
- Height: use `style={{ height: '3px', borderRadius: '999px' }}`
- Color logic:
  - Connector at index `i` connects stop `i-1` to stop `i`
  - If both stop `i-1` and stop `i` are visited (both in `state.visitedStops` and `< state.currentStop`): green `var(--color-lego-green)`
  - If `i === state.currentStop`: red `var(--color-lego-red)` (connector leading INTO current)
  - Else: grey `var(--color-lego-grey-pale)` (upcoming)
- For visited connectors, use the left-to-right sweep via linear-gradient approach:
  ```
  background: linear-gradient(to right, var(--color-lego-green) 50%, var(--color-lego-grey-pale) 50%)
  backgroundSize: '200% 100%'
  backgroundPosition: isVisitedConnector ? 'left' : 'right'
  transition: 'background-position 200ms ease-out'
  ```
  For non-sweep connectors (red/grey), use plain `background: color` with `transition: 'background 200ms ease'`.
