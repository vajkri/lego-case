## Accomplishments
- Replaced old circular numbered progress dots in PresentationFooter with MinifigHead SVG buttons per stop — each head shows state-driven expression (default smile, current grin, visited wink)
- Semantic connector strips between stops: grey (upcoming), red with clip-path sweep wipe (leading to current stop), green crossfade (between visited stops)
- Sub-slide progress dots (SubSlideProgress size="sm") appear only under the current stop; all other stops show a fixed-height spacer to prevent footer reflow
- Hover: scale(1.1) + drop-shadow; Focus: 3px solid #0055BF outline + scale(1.08); neither changes facial expression
- Deleted MapProgressIndicator.tsx — confirmed already unused in MapCanvas; footer is now the sole progress indicator
- 14 new PresentationFooter tests; full suite 151 tests green post-Task 1
- Connector animations reworked after visual verification: clip-path keyframe added to globals.css, green connectors crossfade from red on transition

## Decisions Made
- Red connectors use clip-path inset animation rather than the plan's linear-gradient background-position approach — clip-path gives a sharper, more reliable wipe sweep
- Green (visited) connectors crossfade from red by transitioning the `background` CSS property — simpler than a separate sweep in the opposite direction
- Single `hoveredIndex` / `focusedIndex` (number | null) state variables manage hover/focus for all 5 head buttons — avoids 5 boolean pairs
- Connector animation keyframe placed in globals.css (not inline) to allow CSS animation referencing by name
