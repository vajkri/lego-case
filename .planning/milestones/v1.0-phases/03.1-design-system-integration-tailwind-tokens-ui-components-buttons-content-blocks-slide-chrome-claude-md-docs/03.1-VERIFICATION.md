**Test:** Click the left (grey) and right (red) nav arrow buttons inside the slide overlay.
**Expected:** Each button exhibits the brick press animation — border-bottom collapses from 4px to 2px and button shifts down by 0.5px on click. Both arrows are now Button component instances, so the animation is guaranteed by the same code path as PresentationFooter buttons.
**Why human:** CSS `active:` pseudo-class transitions cannot be tested in jsdom.

**Expected:** Baloo 2 renders for all display text (rounded, playful style), DM Sans for body paragraphs (clean sans-serif). No flash of system font before custom fonts load.
**Why human:** next/font optimization and CSS variable injection require a real browser with network access.
