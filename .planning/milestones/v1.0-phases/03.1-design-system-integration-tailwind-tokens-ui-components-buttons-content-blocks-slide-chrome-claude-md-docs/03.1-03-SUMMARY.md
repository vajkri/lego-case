## Decisions Made

- **variantStyleMap via inline style (not Tailwind):** rgba tints and depth shadows live in CSS `:root`, not `@theme`. Tailwind v4 silently drops rgba values from `@theme`. Inline `style` prop is the correct approach.
- **EntityCards defaults to yellow variant:** Yellow = "what to do next" per color hierarchy; entity cards most naturally use yellow as the recommended default per styleguide.
- **CalloutBox no variant prop:** Always yellow signals it as an important callout; adding variant would dilute this semantic.
- **box-shadow for content block depth:** Preserves rounded corners on block containers. Buttons use border-bottom (no rounded corners to worry about).
