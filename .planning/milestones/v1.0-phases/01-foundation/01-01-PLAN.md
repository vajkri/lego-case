**Step 1 — Create the Next.js app in-place:**

The repo root already exists with CLAUDE.md. Run create-next-app targeting the current directory and pass `--no-tailwind` to avoid v3 being installed automatically:

```bash
npx create-next-app@latest . --typescript --eslint --app --no-tailwind --no-src-dir=false --import-alias="@/*"
```

If prompted whether to install in the current non-empty directory, confirm yes.

**Step 2 — Install required libraries:**

```bash
npm install motion zustand
npm install -D tailwindcss @tailwindcss/postcss postcss
```

Do NOT install `framer-motion` — it is the old package name. Motion 12 is the `motion` package only.

**Step 3 — Configure static export in `next.config.ts`:**

Replace the generated config with:

```typescript
import type { NextConfig } from 'next'

const nextConfig: NextConfig = {
  output: 'export',
  images: { unoptimized: true },  // safety for future phases; no-op in Phase 1
}

export default nextConfig
```

**Step 4 — Create PostCSS config for Tailwind v4:**

Create `postcss.config.mjs` at the repo root:

```javascript
const config = {
  plugins: {
    "@tailwindcss/postcss": {},
  },
}
export default config
```

Delete any existing `postcss.config.js` or `postcss.config.cjs` that create-next-app may have generated.

**Step 5 — Configure Tailwind v4 in globals.css:**

Replace the contents of `src/app/globals.css` with:

```css
@import "tailwindcss";

@theme {
  /* LEGO design tokens deferred to Phase 4 — placeholders only */
  --color-brand: oklch(0.7 0.2 85);
  --font-display: 'Inter', sans-serif;
}
```

Remove all Tailwind v3 directives (`@tailwind base`, `@tailwind components`, `@tailwind utilities`) if present.
