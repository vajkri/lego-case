LEGO Case — Design System

Visual language for the kids.lego.com frontend architecture presentation. LEGO instruction booklet aesthetic: playful, bold, professional. Baloo 2 headings, DM Sans body, single red accent, LEGO brick depth on every interactive element.

Border Widths

Three border tiers. The default 3px border gives cards and blocks a sturdy, tactile feel consistent with the LEGO brick aesthetic. The 4px depth border on buttons creates the "physical press" effect. Content blocks get their depth from a hard box-shadow underneath — like a LEGO plate sitting below the brick.

Depth in Context

Buttons use border-bottom: 4px for press depth. Content blocks use a hard box-shadow (zero blur) — like a LEGO plate sitting underneath the brick. Both approaches create physical depth while preserving clean rounded corners.

Grey Light (#A0A1A3) fails text contrast at 2.6:1. It is restricted to decorative use only: borders, disabled dots, card depth shadows. Never used for readable text. The lightest permissible text color is Grey (#6D6E70) at 5.3:1.

WCAG 2.2 Checklist

1.4.3
Contrast (Minimum) — All text-on-background pairs meet 4.5:1 (normal) or 3:1 (large). Grey Light restricted to decorative.
1.4.4
Resize Text — 16px body minimum. All content text uses relative units (rem/em) in React implementation. No clipping at 200% zoom.
1.4.11
Non-text Contrast — Focus indicators, button borders, and progress dots all meet 3:1 against adjacent colors.
1.4.12
Text Spacing — No fixed-height containers for text. Custom letter/word spacing won't break layouts.
2.4.7
Focus Visible — All interactive elements have visible focus rings (2px solid, offset-2). Blue ring on default, red on active buttons.
2.4.11
Focus Not Obscured — Focus indicators are never covered by overlapping elements. Z-index stacking ensures visibility.
2.5.8
Target Size — Minimum 44×44px for all interactive targets (buttons, stop nodes). Badge/dot progress elements are non-interactive indicators.

Design Principles

The guiding rules that keep every component feeling like it belongs in the same LEGO instruction booklet.

01

Instruction Booklet, Not Software UI

Every element should feel like a LEGO instruction manual. Rounded corners, bold colors, physical depth. The metaphor is the medium.

02

Light & Warm, Not Dark & Techy

Cream and white backgrounds. Warm neutrals. Approachable and inviting — for engineers and business stakeholders alike.

03

Physical Depth on Everything Interactive

Buttons and cards have border-bottom depth. Stud dot patterns add texture. Everything feels tactile — you want to press the buttons.

04

One Accent, Used Consistently

LEGO Red for all chrome. No rainbow per-stop colors. Yellow exclusively for primary action buttons. The hierarchy: red = where you are, yellow = what to do next, green = where you've been.

05

Content-First Slides

White slide backgrounds. Structured content blocks. No decorative clutter competing with information. The LEGO personality comes from typography, chrome, and buttons.

06

Accessible by Default

WCAG 2.2 AA minimum. 16px content text. 44px touch targets. Visible focus rings. No color-only state indicators. Performance budgeted. prefers-reduced-motion respected.

The stud dot pattern is the signature LEGO texture. Used sparingly on headers and hero areas: radial-gradient(circle at 12px 12px, rgba(255,255,255,0.06) 6px, transparent 6px) at 32px 32px. Low opacity adds texture without competing with content.
