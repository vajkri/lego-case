Slide Layout Comparison

Two options for the slide overlay chrome. Same content — different header treatment. Both use the LEGO design system with light theme and stud-pattern header.

A
Minimal Header

Red stud-pattern bar holds only Logo (left) and Close (right). Stop badge, progress dots, and slide heading live inside the content card. Content area has more vertical space for the chrome elements.

Frontend Architecture

Core Approach

The platform will use a modular frontend architecture within a monorepo.

Goals

  • Shared tooling and standards
  • Easy cross-team collaboration
  • Simplified dependency management
  • Clear ownership boundaries

Monorepo Advantages

  • Easier sharing of UI components
  • Consistent CI/CD pipeline
  • Simplified refactoring
  • Unified dependency management
B
Full Chrome Header

Red stud-pattern bar holds Logo (left), stop name + progress (center), and Close (right). The slide content area is purely content — no chrome, no badge. More presentation real estate for the actual slide material.
