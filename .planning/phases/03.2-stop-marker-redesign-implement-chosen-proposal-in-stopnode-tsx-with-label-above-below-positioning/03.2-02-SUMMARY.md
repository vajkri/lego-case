## Decisions Made

- `MarkerPin` removed entirely — no backward compatibility stub needed since BrickMarker is a complete replacement with the same external API
- CSS fill transition on rect `style` prop (not SVG `fill` attribute) — required for CSS transitions to work reliably on SVG elements
- `brick-drop-in` class on topmost brick `<g>` only (not entire SVG) — ensures only the newest brick animates in, not the whole stack
- Label positioning uses `flex-col`/`flex-col-reverse` — simpler and more maintainable than absolute positioning
- React `key` prop on `<g>` wrapper for stud pairs (not fragment) — fragments cannot carry key props in this JSX context
- Road-side placement (-25px left for labels-above, +25px right for labels-below) — resolved overlap with road path
- Brick sizing via SVG width/height at 80% (not CSS transform scale) — CSS scale also scales shadow/filter effects undesirably
- Dynamic SVG viewBox crops to content bounds per stop — eliminates inflated gap caused by unused viewBox space above shorter stacks
- Z-index layering by stop position — stop 3 (label-below, right side) needed to appear in front of car; others behind
- Static label colors: after feedback round 4, labels show only 3 data states (default/active/visited), not 5 visual states
- 80% activation timing (1.12s) — slightly earlier than car arrival creates visual anticipation without jarring effect
