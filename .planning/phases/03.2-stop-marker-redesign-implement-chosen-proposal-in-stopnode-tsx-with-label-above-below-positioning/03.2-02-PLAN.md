  <name>Task 1: Add brick-drop keyframe to globals.css and rewrite StopNode.tsx with BrickMarker</name>

  <action>
    **Part A: globals.css -- Add brick-drop keyframe animation**

    Add to the existing `@layer components` block in `globals.css`:

    ```css
    /* Brick-drop animation -- plays when stop becomes active (Phase 03.2) */
    @keyframes brick-drop {
      from {
        opacity: 0;
        transform: translateY(-10px);
      }
      to {
        opacity: 1;
        transform: translateY(0);
      }
    }

    .brick-drop-in {
      animation: brick-drop 180ms cubic-bezier(0.34, 1.56, 0.64, 1) both;
    }

    @media (prefers-reduced-motion: reduce) {
      .brick-drop-in {
        animation: none;
      }
    }
    ```

    The 180ms duration with spring easing gives a snappy "brick settling" feel. `both` fill mode (shorthand for `forwards`) keeps the brick in final position.

    **Part B: StopNode.tsx -- Complete rewrite**

    Delete the `MarkerPin` component entirely. Create a new `BrickMarker` function component and rewrite `StopNode`.

    **BrickMarker component:**
    - Props: `{ stopIndex: number; state: BrickState }` where `BrickState = 'default' | 'hover' | 'focus' | 'active' | 'visited'`
    - Transcribe ALL coordinate constants from the mockup (POS, STUD, STUD_X, COLORS, PLATE, BRICK_LAYOUTS, ACTIVE_COLORS) as module-level TypeScript constants. Use the EXACT values from the mockup -- do not recalculate.
    - Implement `getColor(level, side)` and `getPlateColor()` functions matching the mockup logic exactly.
    - Render an `<svg>` with `width={VB_W} height={VB_H} viewBox="0 0 48 66" fill="none" overflow="visible" aria-hidden="true"`.
    - Render baseplate: shadow rect at `POS.plate.sy`, face rect at `POS.plate.y`, both `rx="2.5"`.
    - Render level 1 full brick: shadow rect at `POS.level1.sy` (height = `POS.level1.h + 1`), face rect at `POS.level1.y`, both `rx="2"`.
    - Render level 2/3 half bricks conditionally based on `BRICK_LAYOUTS[stopIndex]`. Use `POS.halfL` and `POS.halfR` for x/width. Shadow rect height = `level.h + 1`.
    - Studs: render only on topmost level. Full brick gets 2 studs at `STUD_X.left` and `STUD_X.right`. Half brick gets 1 stud centered. Stud shadow rect: `y = brickY - STUD.shH + 1`, stud face rect: `y = brickY - STUD.h`. Both width = `STUD.w`, no border-radius (`rx="0"` -- sharp rectangles per CONTEXT.md).
    - Apply `style={{ fill: color, transition: 'fill 300ms ease' }}` on ALL `<rect>` elements (not SVG `fill` attribute) to enable color sweep transition from active to visited.
    - Apply CSS class `brick-drop-in` to the topmost brick group `<g>` element ONLY when `state === 'active'`. This triggers the entrance animation.

    **StopNode component rewrite:**
    - Keep: `'use client'` directive, `Stop` type import, `usePresentation` import, `StopNodeProps` interface (unchanged), `handleClick` function (unchanged).
    - Add local state: `const [isHovered, setIsHovered] = useState(false)` and `const [isFocused, setIsFocused] = useState(false)`.
    - Compute visual state with priority: `active > focus > hover > visited > default`:
      ```
      const variant: BrickState = isActive ? 'active' : isFocused ? 'focus' : isHovered ? 'hover' : isVisited ? 'visited' : 'default'
      ```
    - Keep `data-state={variant}` on the button (existing tests depend on it, plus DOM debuggability).
    - Button wrapper structure:
      ```
      <button> (absolute positioned, aria-label, data-state, onClick, onMouseEnter/Leave, onFocus/onBlur)
        <div> (flex column or column-reverse based on labelPosition)
          <span> (label -- always visible)
          <div> (marker wrapper -- receives transforms + outline)
            <BrickMarker />
          </div>
        </div>
      </button>
      ```
    - Label positioning: inner `<div>` uses `flex-col` when `stop.labelPosition === 'above'`, `flex-col-reverse` when `'below'`. This naturally places the label above or below the brick marker.
    - Button styles: `position: absolute`, `left/top` from `stop.coordinates`, `transform: translate(-50%, -100%)` to anchor baseplate bottom at the coordinate point. `background: transparent`, `border: 0`, `padding: 4px`, `cursor: pointer`. NO outline on the button itself -- outline goes on the marker wrapper.
    - Marker wrapper div:
      - Base: `transition: all 0.25s ease`, `transform-origin: center bottom` (so scale grows upward from baseplate).
      - Hover state: `transform: scale(1.1)`, `filter: drop-shadow(0 4px 10px rgba(0,0,0,0.18))`.
      - Focus state: `outline: 3px solid #0055BF`, `outlineOffset: 4px`, `borderRadius: 4px`, `transform: scale(1.08)`.
      - Active state: `transform: translateY(-3px)`, `filter: drop-shadow(0 5px 12px rgba(0,0,0,0.22))`.
      - Visited + default: no transform, no filter.
      - Apply these via inline `style` object computed from `variant`, not Tailwind classes (the specific values like `#0055BF` and `drop-shadow` filter values are not Tailwind tokens).
    - Label styles (per state, matching mockup):
      - Default: `background: rgba(160,161,163,0.15)`, `color: #6D6E70`
      - Hover: `background: rgba(160,161,163,0.22)`, `color: #6D6E70`, `fontWeight: 700`
      - Focus: `background: rgba(0,85,191,0.1)`, `color: #0055BF`, `fontWeight: 700`
      - Active: `background: #F5CD2D`, `color: #2C2C2C`, `fontWeight: 700`
      - Visited: `background: rgba(0,168,80,0.12)`, `color: #00A850`
      - Common: `fontFamily` from `font-display` (Baloo 2), `fontSize: 14px`, `fontWeight: 600` (default), `whiteSpace: nowrap`, `padding: 3px 12px`, `borderRadius: 8px`, `transition: all 0.25s ease`.
      - Apply via inline `style` object.
    - Event handlers on button:
      - `onMouseEnter={() => setIsHovered(true)}`
      - `onMouseLeave={() => setIsHovered(false)}`
      - `onFocus={() => setIsFocused(true)}`
      - `onBlur={() => setIsFocused(false)}`
    - Remove ALL old focus-visible ring Tailwind classes (`focus-visible:outline-none`, `focus-visible:ring-*`). The new focus indicator is a CSS outline on the marker wrapper, not a Tailwind ring on the button.
    - Keep `aria-label` on the button: ``{`${stop.label} -- stop ${index + 1} of 5`}``.

    **Critical implementation notes (from RESEARCH.md pitfalls):**
    - Set `overflow="visible"` on the `<svg>` element so drop-shadow and scale extend beyond bounding box.
    - Set `transform-origin: center bottom` on the marker wrapper so scale grows upward from baseplate.
    - Use `style={{ fill: ..., transition: 'fill 300ms ease' }}` on SVG rects (not SVG `fill` attribute) so CSS transitions work reliably.
    - The brick-drop animation class goes on a `<g>` wrapping only the topmost bricks + their studs, not the entire SVG.
    - Do NOT use the ui `Button` component -- raw `<button>` element stays (locked decision).
    - Keep `data-state` on the button element.
