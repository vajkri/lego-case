**Expected:** Red connector (leading to current) sweeps in left-to-right on mount via `clip-path` animation. When stop 0 becomes visited, the connector crossfades from red to green.
**Why human:** CSS keyframe animation timing and visual quality cannot be asserted programmatically.

#### 3. Hover and focus interactive overlays

**Test:** Hover each minifig head button in the footer; then tab through using keyboard.
**Expected:** Hover: head scales to 1.1x with a soft drop-shadow. Focus: 3px solid #0055BF outline at 4px offset, scale 1.08. Neither hover nor focus changes the facial expression.
**Why human:** Mouse/keyboard interaction and visual overlay rendering require browser verification.

#### 4. Sub-slide dot animation during slide navigation

**Test:** While in slide view (press a head to open), observe the small dot row under the current stop's head. Advance sub-slides with ArrowRight.
**Expected:** Dots appear only under the current stop. Active dot animates (spring) to a pill shape. Other stops show no dots — only the fixed-height spacer.
**Why human:** Motion/react spring animation quality requires live browser verification.

#### 5. MapProgressIndicator removed from map view

**Test:** Open the map view. Check bottom-right corner area and full map viewport.
**Expected:** No progress indicator dots or overlays visible on the map canvas itself. Footer is the sole progress indicator.
