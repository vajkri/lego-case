**Content block components** — all are pure presentational (no 'use client' needed unless they have interactivity — none do). Each component:
- Imports `BlockVariant`, `variantStyleMap` from `./variants`
- Applies variant styles via `style={variantStyleMap[variant]}` on the outer wrapper
- Uses Tailwind classes for padding, border-radius (`rounded-md`), layout, typography
- Uses `font-display` for headings, `font-body` for body text (these are Tailwind utility classes from @theme tokens)

The new section should document (for future Claude instances working on Phase 4+ slides):

**### Design Tokens**
- All tokens are in `src/app/globals.css` @theme block
- Color tokens: `bg-lego-red`, `text-lego-dark`, `border-lego-grey-pale`, etc. (Tailwind utilities auto-generated from `--color-*` tokens)
- Font utilities: `font-display` (Baloo 2) and `font-body` (DM Sans)
- Radius utilities: `rounded-sm` (6px), `rounded-md` (10px), `rounded-lg` (12px), `rounded-xl` (16px), `rounded-pill` (999px)
- Shadow utilities: `shadow-sm`, `shadow-md`, `shadow-lg`
- rgba-based values (tints, depth shadows) are plain CSS custom properties in `:root`, NOT in @theme. Reference via `style={{ background: 'var(--lego-red-tint)' }}` or `[background:var(--lego-red-tint)]`.

**### UI Components**
- `Button`: 3 variants (yellow/red/grey), 2 sizes (icon 48x48 / label h-48). Import from `@/components/ui`. Has brick-depth press effect.
- `SlideFrame`: Chrome wrapper for slides. Pass `onClose`, `leftArrow`, `rightArrow`, `children`. Renders stud-pattern header bar + grey-wash outer + white inner card.
- `StopBadge`: Red pill showing "STOP N . LABEL". Props: `stopIndex`, `stopLabel`.
- `SubSlideProgress`: Dot row for sub-slide navigation. Props: `total`, `current`.
- Content blocks (6 types): `BulletList`, `TwoColumnCards`, `EntityCards`, `NumberedSteps`, `CalloutBox`, `DataTable`. Import from `@/components/ui`.

**### Variant System**
- Content blocks support `variant` prop: `'default' | 'red' | 'yellow'`
- Controls: background tint, border color, depth shadow
- Styles applied via `style` prop (rgba values from :root CSS vars), NOT Tailwind classes
- CalloutBox is always yellow (no variant prop)

**### Typography Scale**
- Document the key pairings from styleguide: Page Title (40px/800), Slide Heading (30px/700), Section Heading (22px/700), Card Title (18px/700), Body (16px/400), Badge (14px/700)
- Minimum content font: 16px. Badge/label exception: 14px.

**### Depth System**
- Buttons: `border-bottom: 4px` for press depth
- Content blocks: `box-shadow: 0 6px 0 0 <color>` (NOT border-bottom — preserves rounded corners)

**### Color Hierarchy**
- Red = "where you are" (current state)
- Yellow = "what to do next" (primary action)
- Green = "where you've been" (visited)
- Grey = "not yet / go back" (secondary)
